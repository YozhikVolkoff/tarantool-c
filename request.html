Building a request with “tnt_stream”¶

Functions in this section add commands by appending them to a tnt_stream buffer. To send the buffered query to the server, you can use the tnt_flush() function and then call tnt->read_reply(), for example.

Adding requests (except UPDATE)¶

ssize_t tnt_auth_raw(struct tnt_stream *s, const char *user, int ulen, const char *pass, int plen, const char *base64_salt)¶
ssize_t tnt_auth(struct tnt_stream *s, const char *user, int ulen, const char *pass, int plen)¶
ssize_t tnt_deauth(struct tnt_stream *s)¶

Two functions for adding an authentication request. ulen and plen are the lengths (in bytes) of user’s name user and password pass.

tnt_deauth is a shortcut for tnt_auth(s, NULL, 0, NULL, 0).

Add a select request.

offset is the number of requests to skip.

iterator is the iterator type to use.

Adding an UPDATE request¶

ssize_t tnt_update(struct tnt_stream *s, uint32_t space, uint32_t index, struct tnt_stream *key, struct tnt_stream *ops)¶

Basic function for adding an update request.

ops must be a container gained with the tnt_update_container() function.

Operations¶

ssize_t tnt_update_bit(struct tnt_stream *ops, uint32_t fieldno, char op, uint64_t value)¶

Function for adding a byte operation.

Possible op values are:

  • '&' - for binary AND
  • '|' - for binary OR
  • '^' - for binary XOR

ssize_t tnt_update_arith_double(struct tnt_stream *ops, uint32_t fieldno, char op, double value)¶

Three functions for adding an arithmetic operation for a specific data type (integer, float or double).

Possible op’s are:

  • + - for addition
  • - - for subtraction
